# 游戏页面推荐游戏

基于当前技术栈（HTML5 Canvas + JavaScript），以下是按难度和类型分类的推荐游戏。

## 🎮 推荐游戏列表

### 🟢 简单级（1-2小时实现）

#### 1. 打砖块 ✅ 已完成
- **难度**：⭐
- **类型**：反应类
- **核心机制**：挡板反弹球消除砖块
- **关键技术**：Canvas 绘图基础, 碰撞检测 (AABB), 动画循环
- **特色**：经典游戏，视觉效果好，易于扩展
- **扩展**：不同类型砖块、道具、关卡系统

#### 2. 2048 ✅ 已完成
- **难度**：⭐
- **类型**：益智类
- **核心机制**：滑动合并数字达到2048
- **关键技术**：二维数组操作, 触摸事件处理, 状态合并逻辑
- **特色**：逻辑清晰，无需复杂动画
- **扩展**：不同数字目标、计分系统

#### 3. 记忆翻牌 ✅ 已完成
- **难度**：⭐
- **类型**：记忆类
- **核心机制**：翻开卡片找到配对
- **关键技术**：DOM 操作/状态管理, 数组洗牌 (Fisher-Yates), CSS 3D 变换
- **特色**：适合所有年龄，可自定义图片
- **扩展**：计时、步数统计、难度等级

#### 4. 猜数字
- **难度**：⭐
- **类型**：逻辑类
- **核心机制**：猜1-100之间的数字
- **关键技术**：基础逻辑判断, 表单交互, 随机数生成
- **特色**：最简单实现，适合初学者
- **扩展**：提示系统、次数限制

#### 5. 井字棋 (Tic Tac Toe) ✅ 已完成
- **难度**：⭐
- **类型**：策略类
- **核心机制**：三子连线获胜
- **关键技术**：胜负判断算法, 状态机, 简单的极大极小值 AI
- **特色**：逻辑简单，适合练习 AI 基础
- **扩展**：人机对战、多人在线、3D 井字棋

#### 6. Pong (乒乓球)
- **难度**：⭐
- **类型**：反应类
- **核心机制**：控制挡板接球
- **关键技术**：简单的 AI 对手逻辑, 碰撞反弹角度计算, 键盘事件
- **特色**：电子游戏鼻祖，极简风格
- **扩展**：双人对战、多球模式、特殊道具

### 🟡 中等级（2-4小时实现）

#### 7. 俄罗斯方块 ✅ 已完成
- **难度**：⭐⭐
- **类型**：策略类
- **核心机制**：堆叠方块完成行消除
- **关键技术**：矩阵旋转算法 (SRS), 碰撞检测, 计时器循环管理
- **特色**：经典游戏，高可玩性
- **扩展**：不同方块形状、等级加速、幽灵方块 (Ghost Piece)

#### 8. 打地鼠
- **难度**：⭐⭐
- **类型**：反应类
- **核心机制**：地鼠随机出现，点击得分
- **关键技术**：多计时器管理 (setInterval), 事件委托, Sprite 动画切换
- **特色**：简单有趣，适合放松
- **扩展**：不同地鼠类型、连击系统、道具

#### 9. 弹球台
- **难度**：⭐⭐
- **类型**：物理类
- **核心机制**：控制挡板保持球运动
- **关键技术**：基础物理模拟 (重力/摩擦力/弹性), 向量运算, 边界检测
- **特色**：物理模拟，视觉效果好
- **扩展**：不同挡板、障碍物、得分区域

#### 10. 扫雷
- **难度**：⭐⭐
- **类型**：逻辑类
- **核心机制**：根据数字推断地雷位置
- **关键技术**：递归算法 (洪泛填充 Flood Fill), 二维网格逻辑, 鼠标右键事件
- **特色**：经典益智游戏
- **扩展**：不同难度、计时、标记功能

#### 11. 五子棋 (Gomoku)
- **难度**：⭐⭐
- **类型**：策略类
- **核心机制**：五子连线获胜
- **关键技术**：极大极小值算法 (Minimax) 初步, 棋型模式匹配, 评估函数设计
- **特色**：规则简单但 AI 深度大
- **扩展**：禁手规则、悔棋功能、复盘模式

### 🟠 进阶级（4-8小时实现）

#### 12. 太空侵略者
- **难度**：⭐⭐⭐
- **类型**：射击类
- **核心机制**：飞船射击外星人
- **关键技术**：对象池模式 (Object Pooling), 简单的粒子系统, 敌人编队移动逻辑
- **特色**：经典街机游戏，可扩展性强
- **扩展**：不同敌人类型、武器升级、Boss战

#### 13. 打飞机
- **难度**：⭐⭐⭐
- **类型**：射击类
- **核心机制**：飞机躲避障碍射击敌人
- **关键技术**：无限滚动背景, 复杂碰撞检测, 触摸/鼠标跟随控制
- **特色**：动作流畅，视觉效果好
- **扩展**：多种敌机、道具、关卡系统

#### 14. 跳跃游戏
- **难度**：⭐⭐⭐
- **类型**：动作类
- **核心机制**：角色跳跃躲避障碍
- **关键技术**：重力模拟, 平台随机生成算法, 摄像机跟随 (Camera Follow)
- **特色**：经典平台跳跃玩法
- **扩展**：二段跳、收集物品、关卡设计

#### 15. 泡泡龙
- **难度**：⭐⭐⭐
- **类型**：策略类
- **核心机制**：发射泡泡消除同色泡泡
- **关键技术**：广度优先搜索 (BFS) 查找连通块, 角度计算 (Math.atan2), 蜂窝网格坐标系
- **特色**：策略性强，视觉效果好
- **扩展**：不同泡泡类型、特殊道具、关卡

#### 16. 水果切切乐 (Fruit Slicer)
- **难度**：⭐⭐⭐
- **类型**：动作类
- **核心机制**：划屏切割飞出的水果
- **关键技术**：鼠标/触摸轨迹追踪, 几何切割算法 (线段交点), 物理抛物线运动
- **特色**：爽快感强，适合移动端
- **扩展**：炸弹障碍、连击加分、3D 效果模拟

### 🔴 高级（8+小时实现）

#### 17. Flappy Bird
- **难度**：⭐⭐⭐⭐
- **类型**：技巧类
- **核心机制**：控制小鸟飞行躲避管道
- **关键技术**：旋转碰撞检测 (OBB), 物理加速度模拟, 视差滚动背景
- **特色**：简单但极具挑战性
- **扩展**：不同障碍、角色皮肤、成就系统

#### 18. 贪吃蛇++ ✅ 已完成
- **难度**：⭐⭐⭐⭐
- **类型**：策略类
- **核心机制**：增强版贪吃蛇，包含复杂元素
- **关键技术**：A* 寻路算法 (AI 对手), 队列数据结构, 迷宫生成算法
- **特色**：迷宫、多重蛇、道具
- **扩展**：迷宫地图、AI对手、道具系统

#### 19. 推箱子
- **难度**：⭐⭐⭐⭐
- **类型**：益智类
- **核心机制**：推动箱子到目标位置
- **关键技术**：深度优先搜索 (DFS) 求解提示, 撤销/重做栈 (Stack), 关卡数据压缩 (RLE)
- **特色**：经典益智游戏，需要策略
- **扩展**：关卡编辑器、撤销功能、多关卡

#### 20. 三消游戏
- **难度**：⭐⭐⭐⭐
- **类型**：益智类
- **核心机制**：交换相邻元素消除三个或更多
- **关键技术**：匹配算法 (Flood Fill 变体), 动画状态机管理, 连锁消除逻辑
- **特色**：类似Candy Crush，高人气
- **扩展**：特殊道具、连击系统、关卡设计

#### 21. 塔防 (Tower Defense)
- **难度**：⭐⭐⭐⭐⭐
- **类型**：策略类
- **核心机制**：建造炮塔阻止敌人到达终点
- **关键技术**：路径规划 (Dijkstra/A*), 行为树 (Behavior Tree), 范围检测与索敌逻辑
- **特色**：系统复杂，成就感强
- **扩展**：多路径、技能树、英雄单位

## 💡 实现建议

### 📂 推荐项目结构

```
game-project/
├── index.html          # 游戏入口
├── css/
│   ├── style.css       # 通用样式
│   └── game.css        # 游戏特定样式
├── js/
│   ├── main.js         # 入口文件
│   ├── game.js         # 游戏循环与状态管理
│   ├── config.js       # 游戏配置参数
│   ├── entities/       # 游戏实体
│   │   ├── player.js
│   │   └── enemy.js
│   ├── systems/        # 游戏系统
│   │   ├── input.js    # 输入处理
│   │   └── physics.js  # 物理/碰撞系统
│   └── utils/          # 工具函数
│       └── audio.js    # 音效管理
└── assets/             # 资源文件
    ├── images/
    └── sounds/
```

### 技术考虑
- **Canvas 绘图**：大部分游戏可用 Canvas 实现，高性能渲染
- **游戏循环**：使用 `requestAnimationFrame` 保证流畅度
- **状态管理**：明确区分 Start, Playing, Paused, GameOver 状态
- **响应式设计**：Canvas 需动态适配 `window.innerWidth/Height`

### 代码复用与优化
- **Entity-Component-System (ECS)**: 考虑简单的 ECS 架构解耦数据与逻辑
- **对象池 (Object Pool)**: 复用子弹、粒子等高频创建销毁的对象
- **离屏渲染 (Off-screen Canvas)**: 预渲染复杂静态背景

## 🔧 技术要求

### 通用要求
- HTML5 Canvas 用于渲染
- JavaScript (ES6+) 用于游戏逻辑
- CSS3 用于 UI 样式与过渡

## 📝 实现清单

每个游戏确保：
- [ ] **核心机制**: 游戏玩法完整实现，无严重 Bug
- [ ] **状态管理**: 包含 菜单 -> 游戏 -> 结束 -> 重来 的完整流程
- [ ] **UI 反馈**: 分数实时更新，明显的胜负提示
- [ ] **交互体验**: 键盘/鼠标/触摸操作灵敏，支持暂停
- [ ] **适配性**: 在不同尺寸屏幕下 Canvas 显示正常
- [ ] **性能**: 稳定 60 FPS，无内存泄漏
- [ ] **代码规范**: 变量命名清晰，关键逻辑有注释
- [ ] **资源**: 图片/音效加载处理，无资源丢失
